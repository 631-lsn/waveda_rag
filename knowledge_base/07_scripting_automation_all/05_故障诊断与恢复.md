# WavEDA 自动化故障诊断与恢复

## 为什么循环只运行一次

常见原因不是 `for` 循环错误，而是第一轮导出失败后代码立即 `load` 不存在的文件，MATLAB 抛异常并终止整个脚本。正确顺序是：

1. 启动 WavEDA。
2. 等待或超时。
3. 验证文件。
4. 文件无效则记录失败并 `continue`。
5. 文件有效才读取和分析。

## 结果节点诊断

节点名称存在不等于节点数据有效。使用以下阶梯：

1. 仿真不导出。
2. 只导出 port。
3. 只导出 observer。
4. 组合导出。

如果仿真通过而某类导出稳定失败，优先在 GUI 删除并重建该结果节点，保存为新的工程版本后单独验证。

## 网格和旧状态

参数变化但曲线完全重合时检查：

- 变量是否参与几何。
- 是否删除旧网格。
- 是否错误复用旧文件。
- 是否加载了错误工程路径。

正式批量前对扫描两端强制重建网格。端点仍无差异时暂停任务。

## 路径迁移清单

- `.tsp` 工程路径。
- WavEDA 可执行文件路径。
- 生成 XML 路径。
- observer/port 输出路径。
- 结果目录和临时目录。

只修改其中一个路径可能导致“启动正常但加载旧工程”或“结果写到另一目录”。

## 资源释放

- 每轮关闭工程和 GUI。
- 点间增加短等待。
- 第二点快速闪退时先确认上一 WavEDA 进程已结束。
- 许可证不支持多实例时禁止并行。

## Bug 复现包

稳定复现脚本接口崩溃时，打包：最小 `.tsp`、触发问题的单个 XML、控制脚本、预期输出名称、退出码十进制和十六进制、GUI 手工执行是否成功、WavEDA 版本、复现步骤和截图。

## WavEDA 退出码参考

退出码不可靠，仿真成功也可能返回非零值。**以数据文件内容为准。**

| 退出码 | 含义 |
|--------|------|
| `0` | 正常 |
| `-1` | 常见，可能是 `close gui` 导致，仿真本身通常成功 |
| `-1073741819` | C++ 内存访问违规（0xC0000005） |
| `-532265403` | C++ 异常 |

```matlab
% 正确判断方式
if exist(port_file, 'file') ~= 2
    fprintf('文件未生成，重试\n');
end
% 不要: if status ~= 0; error('失败'); end
```

## License 偶发失败

WavEDA 偶发报 License 错误，重试后恢复，无法稳定复现。重试机制已覆盖此情况。若频繁出现（每 10 次中 >3 次），联系技术支持检查 License 服务。

## 数据异常检测

某组参数可能物理求解失败但 WavEDA 不报错退出。检测方法：

```matlab
s11_db = port_data(:, 2);
if all(isnan(s11_db)) || any(isinf(s11_db))
    fprintf('[WARN] NaN/Inf, skip\n'); continue;
end
if all(s11_db == 0)
    fprintf('[WARN] all zero, skip\n'); continue;
end
if all(s11_db > 0)
    fprintf('[WARN] all positive, suspicious\n'); continue;
end
```

异常参数记录到数组，循环结束后打印列表。

## 路径含特殊字符

中文、空格等特殊字符在 `system()` 传参时可能编码不一致。路径尽量使用英文和数字。含空格时用双引号包裹。不确定时在 cmd 中手动验证。

## Observer 列数不匹配

代码硬编码读取 3 列但实际文件只有 2 列时崩溃。用 `size(data, 2)` 检测列数后按需读取。

## legend 崩溃

部分参数组跑失败导致对应 legend 为空值。用动态 cell 只保留非空条目：`valid_legend{end+1} = ...`
