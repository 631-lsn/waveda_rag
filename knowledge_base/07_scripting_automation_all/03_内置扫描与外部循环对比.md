# WavEDA 内置参数扫描与外部循环对比

## 两种模式

| 项目 | 外部循环 | 工程内置扫描 |
| --- | --- | --- |
| 参数控制 | MATLAB/Python/Agent | `.tsp` 内参数扫描配置 |
| WavEDA 启动次数 | 每个参数点一次 | 通常一次 |
| XML `modify` | 有 | 通常无 |
| 断点续跑 | 容易逐点实现 | 依赖工程和整体结果格式 |
| 单点重试 | 容易 | 较难 |
| 输出解析 | 每点独立，稳定 | 可能为多列或多块，需专门解析 |
| 当前团队验证 | Dipole/AMC 已验证 | 尚需稳定工程专项验证 |

## Agent 选择规则

- 默认选择外部循环，便于诊断、重试和结果隔离。
- 用户已有稳定内置扫描工程，并希望减少 GUI 启动次数时，才建议内置扫描。
- 不要同时让工程内置扫描和外部循环扫描同一个变量，避免值被覆盖或结果维度失控。

## 内置扫描的最小 XML

```xml
<script version="1.0">
    <command cmdType="load" obj1="project" file="&lt;PROJECT_FILE&gt;" />
    <command cmdType="delete" obj1="mesh" />
    <command cmdType="sim" solver="auto" />
    <command cmdType="export" obj1="result" obj2="port"
             name="Port_S_data_1" file="&lt;PORT_FILE&gt;" />
    <command cmdType="close" obj1="project" />
    <command cmdType="close" obj1="gui" />
</script>
```

## 验证要求

在把内置扫描模板交给用户前，必须用一个真实工程确认：

1. WavEDA 是否一次完成全部组合。
2. 导出文件的列/块如何映射到参数值。
3. 失败时能否识别具体参数组合。
4. 结果节点是否支持该扫描维度。

在这些问题未验证前，标记为 `partial`，不能声称与外部循环同等稳定。
