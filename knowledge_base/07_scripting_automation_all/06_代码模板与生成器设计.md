# WavEDA 自动化代码模板与生成器设计

## 模板不能只做字符串替换

合格模板必须包含：配置、XML 生成、启动超时、旧文件清理、结果验证、重试、进度保存和汇总。仅用 `system()` 后检查 `exist(file)` 不足以交给 Agent 自动运行。

## MATLAB 主控结构

```matlab
% 1. 用户配置
% 2. 工程与输出验证
% 3. 预分配状态和结果
% 4. for 参数点
%    删除本轮旧输出
%    写 XML
%    带超时启动 WavEDA
%    等待文件刷新
%    校验列数、数值和频率
%    保存 progress.csv 和 results.mat
% 5. 生成叠加图和趋势图
% 6. 本地辅助函数
```

推荐使用可终止的进程对象或等价超时机制，不使用无限阻塞的 `system()` 作为唯一实现。

## Python 主控结构

```python
completed = subprocess.run(
    [waveda_exe, f"script={xml_file}"],
    timeout=timeout_seconds,
    capture_output=True,
    text=True,
)
```

还应处理 `TimeoutExpired`，并验证输出文件，而不是只看 `returncode`。

## 单一配置源

XML、MATLAB、Python 和 README 必须从同一个 `task_config.json` 渲染。任务配置应记录：

- 路径。
- 变量和实际值数组。
- 频率单位和目标频率。
- 结果节点与曲线顺序。
- 强制重建网格。
- 超时、重试和等待。
- 结果指标与图表类型。

## 模板质量门槛

- 路径含空格时仍可运行。
- XML 特殊字符已转义。
- 参数文件名处理负号和小数点。
- 重试前删除旧输出。
- 失败点不会终止整个批次。
- 进度每点落盘。
- MATLAB 表格列数与名称一致。
- Python 通过语法检查。
- 生成器支持 `dry-run`：只生成文件和清单，不启动 WavEDA。

## 双参数输出

除了长表 CSV，还可生成热力图和三维曲面，但图形是后处理，不应耦合到仿真成功判定。缺失点保留为 NaN。

## 完整代码模板

本目录 `templates/` 下提供可直接运行的完整代码（含完整 MATLAB/Python 源码和 XML 模板）：

```
templates/
├── 01_single_param/          ← 单参数扫参（XML 模板 + modify_script + 主控）
├── 02_dual_param/            ← 双参数扫参（n×m 嵌套循环 + 2D/3D 图）
├── 03_builtin_sweep/         ← 工程内置扫描（一次运行，多列读取）
├── 04_python/                ← Python 版单参数扫参
├── 05_direct_tsp_tools/      ← 直接改 .tsp 的 8 个独立工具（实验性）
└── 06_direct_tsp_sweep/      ← 直改 .tsp 扫参（精简 XML 模板 + modify_tsp）
```

每个模板文件包含完整可执行代码，可直接复制或用生成器参数化。
