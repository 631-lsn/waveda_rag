# WavEDA 导出结果解析与成功判据

## 核心原则

WavEDA 退出码只用于诊断。真实成功必须由本轮导出文件证明。

```text
成功 = 文件存在
    且属于本轮任务
    且大小合理
    且可以读取数值
    且频率/时间范围合理
    且所需曲线完整
```

## 运行前处理

- 删除本轮目标文件，防止旧文件假成功。
- 为参数值生成唯一文件名。
- 保存任务签名：工程哈希或修改时间、变量、值、频扫、结果节点、曲线顺序、是否重建网格。

## 最低内容校验

- 非空且存在足够数值行。
- 不能全为 NaN、Inf 或零。
- 频率列应递增；堆叠曲线允许在新块开始时重置。
- 点数和范围应与工程频扫大致一致。
- 需要 S11/S21/相位时必须确认曲线数，不能只检查文件存在。

## 常见布局

### 共用频率列

```text
frequency  S11  S21
```

### 成对列

```text
frequency1  S11  frequency2  S21
```

### 两列堆叠块

```text
frequency  S11
...
frequency  S21
...
```

曲线顺序应来自任务配置，不要只靠变化范围猜测。

## 断点续跑

只有结果文件和任务签名同时匹配时才能复用。模型、频率步长、节点、曲线顺序或网格策略改变后，应使用新输出目录或将旧结果标为过期。

## 指标注意事项

- 最小 S11 位于扫频边界时标记“边界最小值”。
- AMC 相位先展开，再计算 0°、+90°、-90° 交点和带宽。
- 模式转换器分析 S11、目标模式 S21、插入损耗和带内最差 S21。
- 目标频率不是实际求解点时允许插值，但报告必须注明。
