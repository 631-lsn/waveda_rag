# WavEDA 单参数与双参数扫描策略

## 单参数外部循环

适合首次验证和灵敏度分析。每个值生成独立 XML、原始结果和状态记录。

```text
参数值 → 生成 XML → 删除旧输出 → 启动 WavEDA
      → 等待/超时 → 校验结果 → 保存进度 → 下一值
```

用户给范围和步长时生成等步长数组；用户给点数时生成 `linspace`；用户给离散值时保持其顺序。

## 双参数外部循环

两个变量全部组合时，仿真次数为 `N1 × N2`。Agent 必须在运行前显示总次数和预计时间。

```xml
<command cmdType="modify" obj1="var" name="w2" value="0.70" />
<command cmdType="modify" obj1="var" name="L_t2" value="7.5" />
<command cmdType="delete" obj1="mesh" />
```

全部变量修改后只删一次网格、只仿真一次。推荐输出：

- 每个组合的原始文件。
- 长表 CSV：变量1、变量2、状态、目标指标。
- 目标指标热力图。
- 数据足够规则时生成三维曲面。

## 粗到细策略

仿真成本高时优先：

1. 5–10 点粗扫。
2. 找出满足约束或目标较优的区间。
3. 在局部区间加密。
4. 对最优点和相邻点强制重建网格复算。

## 变量合法性

- 只直接扫描基础变量，不直接扫描由其他变量构成的方程变量。
- 检查长度、厚度、间隙和半径是否保持正值。
- 检查总长度、端口位置和计算区域是否会随变量联动。
- 扫描范围可能造成几何自交时必须向用户确认。

## 并行限制

默认串行。不要直接把循环改为 `parfor` 或同时启动多个 WavEDA：许可证数量、GUI 单实例限制和共享求解资源都可能失败。只有在许可证和多实例经过单独验证后，才允许用户显式开启有限并发。
