# WavEDA 自动化快速开始与策略选择树

## 选择树

```text
用户是否已有参数化 .tsp？
├─ 否：先完成参数化建模和 GUI 单点验证
└─ 是
   ├─ 结果节点是否存在并能在 GUI 导出？
   │  ├─ 否：创建/修复结果节点并另存稳定工程
   │  └─ 是
   ├─ 用户是否给出明确参数值？
   │  ├─ 是：单参数或双参数外部循环
   │  └─ 否：根据目标采用粗到细策略
   └─ 是否已有稳定的工程内置扫描？
      ├─ 是：可评估一次启动模式
      └─ 否：默认外部逐点循环
```

## 第一次运行

1. 用一个非基准参数点生成 XML。
2. 删除旧网格。
3. 仿真并导出一个核心结果。
4. 验证文件时间、数值、频率和曲线数量。
5. 再加入全部结果和批量循环。

## 用户只需要改的配置

- 工程路径。
- WavEDA 路径。
- 输出目录。
- 变量和实际值数组。
- 结果节点名称和曲线顺序。
- 目标指标/频率。

## 运行后应看到

- 每个参数点独立 XML 和原始数据。
- `progress.csv` 持续更新。
- 成功数、失败数、尝试次数和退出码。
- 汇总数据和趋势图。
- 失败时指出是加载、仿真、port 还是 observer 阶段。

如果脚本只给出“找不到结果文件”，应回溯 WavEDA 首个失败命令，而不是只修 MATLAB 的 `load` 行。
