# 手动重建索引按钮设计

## 目标

在当前实际使用的右侧数据侧栏中，为知识库状态卡片增加一个“重建索引”按钮，允许用户在不重启应用的情况下从知识库源文件重新构建索引。

## 交互

- 按钮位于右侧数据侧栏的知识库、知识块、模型、监听状态卡片下方，现有“设置”和“收藏夹”按钮上方。
- 点击按钮复用现有 `_rebuild_async()`，在后台线程调用 `build_knowledge_base(self.settings)`。
- 重建期间沿用现有忙碌遮罩和“正在重建知识库”状态，并禁用提问、导入和重建按钮，避免并发构建。
- 成功后复用 `_on_rebuild_done()` 重新载入 pipeline、更新知识块数量并同步监听快照。
- 失败后复用现有 `error_rebuild_title` 错误提示。
- 增加中英文按钮文字；语言刷新时同步更新按钮文字。

## 实现边界

- 不改变自动监听重建逻辑、索引文件格式或构建管线。
- 不在未点击按钮时触发额外构建。
- 只在当前实际使用的 `_sidebar_panel()` 添加按钮；不修改未被 `_build_ui()` 使用的旧 `_left_panel()`。
- 将按钮暴露为 `rebuild_index_button`，便于界面测试验证文案、信号连接和忙碌禁用状态。

## 验证

- 界面测试验证按钮位于侧栏、显示中英文文案，并点击后调用 `_rebuild_async()`。
- 测试忙碌状态会禁用和恢复重建按钮。
- 运行完整测试集、`compileall` 和 `git diff --check`。
